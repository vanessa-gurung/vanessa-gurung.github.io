Version Control with Git

What's the difference between Git and GitHub?

Git is a software that you download onto your computer and does not require any remote servers to use. It is a DVCS – distributed version control system, which means multiple people can work on a particular project together without overwriting each other’s changes. It also has a recorded history of changes. Github also uses version control, but it is a service a user signs up to. It is a code host that stores and shares code.

Install and Explore Git Reflect

What is a Git workflow?

In a typical Git workflow when working with version control, there are three main areas – the working directory, the staging index and the repository. The git shell command is used to move the files from section to section. Files are uploaded in the working directory. Once worked on, they can be moved to the staging index, where all the files that are about to be committed are situated. Once committed, they move to the repository, where those files are saved.

Track and Commit

How would you describe stage and commit to your non-tech-savvy friend?

These terms are used in a code host software called Git to explain a typical workflow. There are three main areas of this workflow: working directory, staging area and git repository. Files are created or copied to the working directory, where they can be changed. Once they have been worked on, they get moved to the staging area i.e. the files are ‘staged.’ This means they are being prepared and collated for a commit which occurs in the git repository. A commit is like a snap shot of the work’s currently staged changes – it saves the changed files to a git repository, which tracks and saves the history of all changes made to the project.

Branch, Pull, Merge

What is main?

All repos have a main branch, which is where the best version of the working code lives.

Why create a branch?

Creating branches allows you to try out various ideas and ways of coding without affecting the main branch. You can commit changes and experiment using different branches until you’re happy with the changes, whereupon you will let other contributors review it before it is merged with the main.

Do the concepts in intuitive or difficult to understand?

Concepts seem to make sense when learning about them but when they come up later, sometimes I’ll have trouble remembering their exact meaning. I guess somewhat intuitive.
