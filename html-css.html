CSS Concepts

What does it mean to display inline vs. inline-blocks?

Display properties are used in CSS to lay out elements on a webpage. Two widely used display properties are display: inline and display: inline-block. Before explaining what they do respectively, I think it is necessary to explain what block level elements do. Block level elements always stack on top of each other even when there’s enough space to be side by side. The default width is 100% and even when the width is less, a new line of content is forced automatically. This is good for things like paragraphs. The most common block elements are p and div.

Inline:

In contrast to block elements, inline elements display side by side, as they live inside other things and don’t force new lines. The width is determined by the size of its content as opposed to 100%. This is good for when you don’t want a new line of content. However, the limitation of display: inline is that you cannot change the height of the element. Padding and margins still apply, but they cannot be used to change the height visibly. Common inline elements include: a span img and button.

Inline-block:

display: inline-block is a combination block and inline, as the name suggests. We can set the height and width properties as well as place elements side by side. We can also set the margin and padding the same way you could with a block level element, but also allow the elements to not force new lines.

Describe what the DOM is and an example of how you might interact with it.

The Document Object Model helps define the structure of a website through breaking it down into a tree structure composed of nodes and objects that allow for programming languages to interact with the page. There are many ways to interact with the DOM to change aspects of the page. This is done through selecting elements through their tags, id or class. A common way of doing this is with the querySelectorAll method. And then changes can be made to those elements.

Explain the difference between accessing data from arrays and objects.

Both arrays and objects are methods to store data in JavaScript, but they differ in several ways.

An array is a list of items e.g. different types of squid:

let squidTypes = [“Giant”, “Colossal”, “Vampire”, “Humboldt”, “Japanese Flying”];

Each item has an index number assigned to them, starting from 0. “Giant” would be assigned 0, as it is the first item in the array and “Japanese Flying” would be assigned 4, as it is the last item in the array. It is through these indexes in which array items can be accessed e.g.
let squid = squidTypes[2] which would be “Vampire”


Objects similarly list things, but they come in the form of key value pairs:

let cookShowHost = {
readySteadyCook: “Ainsley”,
ironChefAmerica: “The Chairman”,
barefootContessa: “Ina Garten”
};

Object properties can be accessed in two ways using: variable.property; or variable[“property”]; e.g.

let bestCook = cookShowHost.readySteadyCook
let bestHost = cookShowHost["readySteadyCook"]



Explain what functions are and why they are helpful.

Functions are statements used in JavaScript that perform tasks or calculate values. They firstly need to be defined and then called upon. They’re handy because if you know that you you’ll be using it repeatedly, you can just use the function name, rather than repeatedly typing out all the code. This means if you’ve made a mistake and would like to change the code within the function, you only have to change the OG one, rather several.
